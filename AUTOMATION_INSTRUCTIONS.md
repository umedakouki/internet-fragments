# 余白採集室 日次運用

## 開始

1. この指示書と自動化メモを読む。
2. `git status --short --branch`、`data/genres/index.json`、各ジャンルの件数・`family`・履歴、直近の日記を確認する。
3. 未コミット変更を戻さず、直近と同種の作業をなるべく避ける。

## 主作業

毎回、次から1つ選ぶ。

- 新ジャンルを10〜50件で始める
- 既存ジャンルを深掘りする
- ジャンルを分割・統合する
- 標本を移動する
- ジャンルを非公開化する

共通タグ、件数の偏り、異質な `family`、新しい比較軸から判断する。既存棚の追補・移動・再編に最低件数はない。必要なら名称、説明、関連、代表資料、地図配置も直す。UIは不具合がある場合だけ変更する。

## 収集・編集

- 媒体は限定しない。写真、文章、音声、動画、イラスト、ウェブ、コード、データ、メタデータだけの記録も対象。
- ID、原典、作者、記録日、権利、媒体を原典で確認し、重複、リンク切れ、出典不明、スパム、個人情報、危険な資料を除外する。
- APIメタデータは一括、ファイルは直列で取得する。取得済み資産を再利用し、429・一時的5xxは待機を増やして再試行する。TLS検証は無効化しない。
- 再配布できない資料や大容量資料は複製せず、必要最小限の説明・プレビュー・原典リンクだけを保存する。全文転載しない。
- 一時物は `output/` に置いて成功後に削除する。日付別JSONや使い捨てスクリプトを作らない。
- 収集不能なら検索、原因、再試行方法を当日の日記へ `frozen` と記録し、件数を水増ししない。

標本は1つの主ジャンルに置き、横断関係は `tags`、編集関係は `relatedGenres` で表す。`archived` は索引に残して非公開、統合元は `merged` と `redirectTo` を残す。変更はジャンル履歴と当日の日記へ記録する。

更新対象は原則、ジャンルJSON、`data/genres/index.json`、必要資産、ジャンル履歴、当日の日記だけとする。

## 完了

次の1コマンドで全データ、対象原典、PC・390px実ブラウザー、探索操作、URL、commit・push、Pages公開を検証する。

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts/run-automation.ps1 -Genre genre-id -CommitMessage "Describe the change" -Publish
```

公開しない場合だけ `-Publish` を外す。失敗は可能な限り修復し、原因と再試行方法を日記と自動化メモへ残す。最後に意図した変更だけが残っていることを確認する。
